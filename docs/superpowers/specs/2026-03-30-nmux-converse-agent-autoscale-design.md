# Feature B: エージェント自動増設機能 設計書

**作成日**: 2026-03-30
**対象スクリプト**: `~/.nmux/bin/nmux-converse`
**ステータス**: 設計承認済み

---

## 概要

nmux-converse の会話ループ中に、複数条件のスコアリングで「エージェント増設が必要か」を自動判定し、
ユーザの承認を得た上でエージェントを追加する機能。

---

## アーキテクチャ

```
_run_loop
  └─ 各ターン終了後 → _score_scale_up()
       └─ スコア >= 5 → _recommend_scale_up()
            ├─ 追加数をユーザに確認（AI推奨値を提示）
            ├─ 上限数をユーザに確認
            └─ _add_agents(count, max)
                 ├─ ラベル自動採番（agent-c, agent-d...）
                 ├─ tmuxペイン存在確認
                 │   └─ 未存在 → ペイン自動作成
                 └─ nmux-bridge name で登録
```

---

## スコアリング

| 条件 | ポイント | 判定基準 |
|---|---|---|
| タイムアウト発生 | +3 | 直近3ターン中1回以上 |
| エージェント応答失敗 | +2 | 直近3ターン中1回以上 |
| エージェント間負荷不均衡 | +1 | 応答文字数の最大/最小比 > 3 |
| ターン数節目 | +1 | 設定ターン数の50%・75%到達時 |

**閾値**: 合計スコア >= 5 で推奨トリガー

---

## ユーザ対話フロー

```
[自動増設の提案]
現在の会話負荷が高まっています。
エージェントを追加すると効率が上がる可能性があります。

AIの推奨追加数: 2名

何名追加しますか？ (数字を入力): _

最大何名まで許可しますか？ (数字を入力): _
```

- キャンセル: 0 入力または Ctrl+C → 増設しない、会話継続
- 同一セッション中に再トリガーされた場合も毎回確認を取る

---

## エージェント自動命名

既存ラベルを `nmux-bridge list` で取得し、連番で採番。

```
既存: agent-a, agent-b
追加: agent-c, agent-d（count=2の場合）
```

命名規則: `agent-[a-z]` の連番。26個を超える場合は `agent-aa` 以降。

---

## tmuxペイン自動作成

`nmux-bridge list` でラベルが未登録の場合：

1. `tmux split-window -t <現在のセッション>` で新規ペイン作成
2. `nmux-bridge name <pane-id> <label>` で登録
3. ユーザに「ペイン <label> を作成しました」と通知

---

## エラーハンドリング

| エラー | 対応 |
|---|---|
| tmuxセッション外で実行 | ペイン作成をスキップ、手動作成を案内 |
| ラベル採番上限超過 | エラーメッセージ、増設中断 |
| nmux-bridge 登録失敗 | ワーニング表示、該当エージェントをスキップ |
